# Finance Compass 需求与设计文档

本文档记录 `Finance Compass` 当前版本的产品需求、业务规则、数据设计和技术实现。它以个人真实记账和资产监控为核心目标，而不是只做简单流水记录。

## 1. 产品定位

`Finance Compass` 是一个跨平台个人理财 App，目标是帮助用户长期维护自己的财务数据，包含日常现金流、信用卡、预算、资产账户、未来支出和 AI 可读摘要。

核心原则：

- 本地优先：数据默认保存在本机 SQLite 数据库。
- 可追溯：账户余额、预算余额、资产快照都尽量能解释来源。
- 多币种：账户和交易保留原币种，总览和报表按用户设置的主币种换算。
- 计划与实际分离：预计支出用于预留和预测，不直接影响当前账户余额。
- AI 外接：当前版本不直接接入 AI，而是导出结构化 JSON 给 ChatGPT 等工具分析。

## 2. 用户需求

### 2.1 账户管理

用户需要管理多种真实账户：

- 现金
- 银行储蓄户口
- 电子钱包，例如 TNG eWallet
- 信用卡
- 养老金
- 股票账户
- 基金账户
- 加密货币账户
- 交易户口
- 货币基金

账户有两层分类：

- `accountType`：真实账户类型。
- `reportGroup`：报表统计分组。

当前统计口径：

- `cash`：现金、银行储蓄、电子钱包。
- `credit`：信用卡。
- `investment`：股票、基金、交易户口、加密货币、货币基金。
- `retirement`：养老金。

账户页需求：

- 可以新增、编辑账户。
- 有关联交易或快照的账户不应随意删除。
- 账户余额显示应按账户自己的币种显示。
- 主币种汇总时需要按设置汇率换算。
- 可选择统计截止月份。
- 截止月份之后的未来交易不纳入账户余额。
- 可查看账户余额追溯。
- 可标记账户已对账到某个月份。

### 2.2 交易记录

交易类型：

- `income`：收入。
- `expense`：支出。
- `transfer`：账户间转账。
- `adjustment`：注资或投资账户调整。

交易需求：

- 可以新增、编辑、删除交易。
- 可以按月份范围筛选交易，默认当月。
- 可以按账户、类型、类别和状态筛选。
- 账户筛选需要同时包含转出和转入记录。
- 交易列表需要按类型显示汇总。
- 类别和账户选择按名称排序。
- 新增交易默认优先使用常用账户和类别。
- 快速模板默认折叠，点击模板直接打开新增交易。
- 周期交易默认折叠，保留金额显示。
- 周期交易生成时可选择 1-12 个月。

状态语义：

- `planned`：预计交易，用于未来现金流和预算预留，不影响真实账户余额。
- `actual`：已发生交易，会影响账户余额。
- 历史兼容字段 `settled` 在 UI 中等同于 `actual`。

多币种转账：

- 转出金额使用转出账户币种。
- 转入金额使用转入账户币种。
- 用户可输入 “转出多少” 和 “转入多少”，避免跨币种转账金额被错误推算。
- 系统会根据设置汇率自动给出初步估算，但用户可覆盖。

### 2.3 分类管理

分类类型：

- `income`
- `expense`
- `investment`
- `transfer`

需求：

- 用户可以自行新增和编辑分类。
- 分类删除需要保护：已关联交易或预算的分类不可删除。
- 交易筛选时，如果已选择交易类型，类别下拉只显示对应类型的类别。
- 分类显示需要紧凑但可读。

### 2.4 预算管理

预算设计为可复用规则，而不是每个月都必须重新创建。

预算字段：

- 类别
- 生效月份 `monthKey`
- 金额
- 币种
- 警戒比例
- 是否启用 rollover

预算需求：

- 预算页面默认显示当月，可选择未来 6 个月。
- 每个预算卡片默认折叠，只显示余额和预算。
- 展开后显示实际、预计、月预算、年度预算、年内已用和使用进度。
- 单个预算卡片按预算自己的币种显示。
- 总预算汇总按主币种显示。
- 预算需要同时考虑实际支出和预计支出。
- 可以编辑和删除预算。

Rollover 规则：

- 未启用 rollover：每月只使用当月预算。
- 启用 rollover：上月余额会带到下月。
- 正数余额会增加下月可用额度。
- 负数余额也会带到下月，表示上月超支会减少下月额度。

预算余额公式：

```text
本月余额 = 上月结转 + 本月预算 - 实际支出 - 预计支出
```

### 2.5 Dashboard 总览

Dashboard 目标是快速回答：

- 当前筛选期间收入多少？
- 当前筛选期间支出多少？
- 当前结余多少？
- 到所选月份为止的资产状态如何？
- 未来是否有已知支出需要预留？
- 哪些预算类别接近或超过预算？

需求：

- 默认时间范围是当年，默认月份是当月。
- 月度对比只显示选择月份前 2 个月到当前月份，共 3 个月。
- 支持选择年份、月份或全部。
- 期间收入、期间支出、结余按筛选时间段计算。
- 现金、信用、投资、退休按余额口径显示。
- 总资产和净资产按截至所选月份结算。
- 信用卡余额在 Dashboard 中以负债语义处理。
- 预算监控默认当前月；若 Dashboard 筛选为全部，预算仍默认当月。
- 最近交易和未来支出需要紧凑展示。

### 2.6 报表

报表需求：

- 支持收入曲线和支出曲线放在一起。
- 支持单月不累计和累计模式。
- 支持线图、饼图和表格。
- 表格以月份为横轴。
- 支持近三个月、近六个月、近一年、本年度等区间。
- 本年度视图可包含未来月份，并用视觉标记区分未来。
- 月数据为 0 的月份在均值和预测类计算中应省略，避免失真。
- 饼图图例应放在饼图上方，适合手机窄屏。

### 2.7 资产与快照

资产账户包括投资和退休类账户。

需求：

- 可记录资产快照。
- 快照可编辑、删除。
- 资产详情显示累计投入、累计取出、总市值、累计成本、现金余额和未实现盈亏。
- 资产卡片可以显示简单趋势。
- 进入资产详情后显示更完整的累计投入、累计取出、总市值三线趋势。

业务语义：

- 初始快照可保留累计成本，用于迁移已有历史资产。
- 后续新增投入不需要用户手动重新计算累计成本。
- 投入交易会增加总市值、累计投入、累计成本和现金余额。
- 取出交易会增加累计取出，并按业务规则减少相应余额。
- 删除注资记录时，对应增加过的成本、现金余额和市值也需要回退。
- 投资成本和资产计算都应以所选截止月份为准。

未实现盈亏建议公式：

```text
剩余成本 = 累计投入成本 - 累计取出
未实现盈亏 = 当前总市值 - 剩余成本
```

### 2.8 资产目标

需求：

- 可以同时设置多个资产目标。
- 目标以扣除信用后的净资产为判断基础。
- 达成后系统自动记录首次达成日期。
- 资产页面显示所有目标共享的一条资产达成曲线。
- 目标卡片中未达成目标显示在前，已达成目标显示在后。
- 已达成目标显示达成年份月份。

### 2.9 导入导出

导出需求：

- 完整 JSON 导出：用于备份和恢复，应包含数据库所有数据。
- AI 摘要 JSON：用于外部 AI 分析，可以不包含完整交易明细，但要按月份保留足够细粒度。
- 未来规划 CSV：用于类似 spreadsheet 的未来现金流规划。
- 导出时应询问保存位置。
- 导出 AI 摘要后尝试直接打开 JSON，让 Android 或桌面系统弹出可打开该文件的 App。

导入需求：

- JSON 导入前先预览。
- 显示账户、类别、预算、交易、快照数量。
- 导入会覆盖当前资料，需要用户确认。

### 2.10 设置

需求：

- 主题可切换。
- 主题颜色应影响全局字体、卡片和边框。
- 支持自然景观主题，如 Ocean、Sky、Forest、Dune、Aurora 等。
- 可写入示例资料，但默认安装不应有示例数据。
- 汇率优先级可拖拉排序。
- 第一个币种是主币种。
- 第二个币种是辅助提示币种。
- 支持 MYR、TWD、CNY、USD。

## 4. 核心计算规则

### 4.1 账户余额

账户余额按账户币种保存和显示。

跨账户汇总时：

```text
主币种金额 = 原币种金额 * 原币种对主币种汇率
```

未来交易不纳入当前账户计算。

### 4.2 转账

转账不是收入或支出。

转账影响：

- 转出账户扣除 `amount/currency`
- 转入账户增加 `toAmount/toCurrency`

### 4.3 预算

预算监控内部按主币种计算，展示时单个预算可换回预算币种。

### 4.4 预测

储蓄预测和均值计算应忽略没有数据的月份，并以第一个有数据月份作为有效起点。

### 4.5 信用卡

信用卡在账户明细中保留真实余额；在 Dashboard 和净资产计算中按负债语义处理。

## 5. 技术设计

技术栈：

- Flutter
- Dart
- SQLite
- Drift
- `file_picker`
- `file_saver`
- `open_filex`
- Puro 管理 Flutter SDK

项目结构：

- `lib/src/core/data`：Repository 和业务聚合。
- `lib/src/core/database`：Drift 数据库和表结构。
- `lib/src/core/models`：领域模型。
- `lib/src/core/settings`：设置状态。
- `lib/src/core/theme`：主题系统。
- `lib/src/core/utils`：日期、货币、ID 等工具。
- `lib/src/features/dashboard`：总览。
- `lib/src/features/accounts`：账户和资产。
- `lib/src/features/transactions`：交易、模板、周期规则。
- `lib/src/features/budgets`：预算。
- `lib/src/features/reports`：报表。
- `lib/src/features/settings`：设置、导入导出、汇率。
- `lib/src/features/shared`：共用 UI。

## 8. 后续优化路线

P0：

- 稳定账户、交易、预算、报表、导入导出。
- 修正余额、快照、预算和多币种计算缺陷。
- 保持 UI 简洁可用。

P1：

- 周期交易系统化。
- 预计/实际支出分层。
- 预算池和年度预算。
- 信用卡还款提醒。
- 未来现金流预测。

P2：

- 更精细的报表交互。
- 更完整的资产分析。
- 更强的 AI 摘要结构。
- 数据同步或云备份。

非目标：

- 不做自动银行同步。
- 不做证券行情自动拉取。
- 不提供投资买卖建议。
- 投资模块只保留轻量资产跟踪，深入投资分析可作为另一个 App。
